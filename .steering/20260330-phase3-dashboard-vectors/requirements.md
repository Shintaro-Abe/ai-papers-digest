# Phase 3 要求内容 — Web ダッシュボード + セマンティック検索

## 1. 概要

Phase 3 として、以下の2機能を構築する：
- **F10: Web ダッシュボード** — 過去の論文要約を閲覧・検索・フィルタリング
- **F11: セマンティック検索** — S3 Vectors によるベクトル類似度検索で関連論文を発見

Phase 1 で構築した S3 静的詳細ページを拡張し、Phase 2 のフィードバックデータを活用する。

## 2. スコープ

### 対象機能

| # | 機能 | 概要 |
|---|------|------|
| F10 | Web ダッシュボード | S3 静的サイトのトップページ + 日付別一覧 + タグ別一覧 + キーワード検索 |
| F11 | セマンティック検索 | S3 Vectors にベクトル保存 + 類似論文クエリ + 詳細ページに「類似論文」セクション追加 |

### 対象インフラ（新規追加分）

- S3 Vectors（vector bucket + vector index、dimension=1024, cosine）
- Bedrock Titan Embeddings V2（`amazon.titan-embed-text-v2:0`、1024次元）
- ※ Lambda + API Gateway による検索 API は不要（クライアントサイド検索で実現）

### スコープ外

- ユーザー認証（個人利用のため不要）
- サーバーサイドレンダリング（静的サイトで実現）
- リアルタイムダッシュボード（日次バッチ更新で十分）

## 3. ユーザーストーリー

### US4: 過去の要約を検索する
> AIエンジニアとして、過去に配信された要約をキーワードや分野で検索したい。
> それにより、必要な時に特定の論文情報にアクセスできる。

**受け入れ条件:**
- [ ] トップページに最新ダイジェストが表示される
- [ ] 日付選択で過去のダイジェストを閲覧できる
- [ ] タグクリックで同じタグの論文一覧が表示される
- [ ] キーワード検索で過去の要約を検索できる（クライアントサイド全文検索）

### US5: 類似論文を発見する
> AIエンジニアとして、気になる論文の類似論文を見つけたい。
> それにより、関連する研究を効率的に探索できる。

**受け入れ条件:**
- [ ] 各論文の詳細ページに「類似論文」セクションが表示される
- [ ] 類似論文は S3 Vectors のコサイン類似度で上位5件を表示
- [ ] 類似論文のタイトル + 一言サマリー + リンクが表示される

## 4. 受け入れ条件

### AC1: Web ダッシュボード
- [ ] トップページ（index.html）に最新日付のダイジェストが表示される
- [ ] 日付ナビゲーション（前日/次日）で過去の日付に移動できる
- [ ] タグ一覧ページ（tags/index.html）から特定タグの論文を閲覧できる
- [ ] クライアントサイド検索（JavaScript）でタイトル・要約をキーワード検索できる
- [ ] モバイルレスポンシブ対応
- [ ] CloudFront 経由で HTTPS アクセス、Geo 制限 JP

### AC2: セマンティック検索
- [ ] summarizer が要約生成時にベクトル埋め込みを S3 Vectors に保存する
- [ ] 各詳細ページに類似論文 Top 5 が表示される
- [ ] 類似論文データは summarizer のバッチ処理で静的に生成（リアルタイム API 不要）
- [ ] S3 Vectors の距離メトリクスは cosine

### AC3: インフラ
- [ ] S3 Vectors リソースが作成される（Terraform または boto3）
- [ ] Bedrock Titan Embeddings のモデルアクセスが有効化されている
- [ ] 追加月額コストが $1 未満に収まる

## 5. 制約事項

- Web ダッシュボードは **S3 静的サイトのみ** で実現（サーバーサイドなし）
- 検索は **クライアントサイド JavaScript**（lunr.js 等）+ 静的 JSON インデックス
- セマンティック検索の類似論文は **バッチ処理で事前計算** し静的 HTML に埋め込む（リアルタイム API 不要）
- S3 Vectors は **ゼロスケール** で月額 < $0.05
- 埋め込みモデル: Bedrock Titan Embeddings V2（1024次元）。コスト: ~$0.00002/1K tokens
  - ※ Titan V2 は 256, 512, 1024 のみ対応（768 は非対応）

## 6. 前提条件

- Phase 1/2 が稼働中であること
- S3 + CloudFront の静的サイトが動作中であること
- Bedrock Titan Embeddings V2 のモデルアクセスが有効化されていること
- S3 Vectors が ap-northeast-1 で利用可能であること（GA 確認済み）
